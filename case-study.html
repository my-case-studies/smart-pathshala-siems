Engineering Leadership: Scaling Smart Paathshala to 65K+ Users

Smart Paathshala is a full education management system, from attendance, class operations, and exams to billing and payment confirmation. I led the platform as technical lead across architecture, backend optimization, frontend and React Native review, and production operations. The system is designed to be fully customizable and compatible for any school or college with tenant-aware workflows.

Payment Flow Complexity I Led

Bill payments are not processed in our UI. Payment execution happens inside the respective provider apps: bKash, Rocket, Nagad, and Tap.

Our backend infrastructure handles end-to-end orchestration: request validation, transaction pipeline handling, settlement-state processing, and confirmation lifecycle back into institutional billing flows.

This design is significantly more complex than normal payment gateway integration because it requires resilient multi-rail state management, strict reconciliation behavior, and high-trust operational consistency.

How I Designed for Scalability and Backend Optimization

I optimized backend development for performance, stability, and maintainability, not only feature speed. Every architecture decision was tied to scaling behavior and production reliability.

Multi-Tenant Data Model Primary and secondary MongoDB strategy with dynamic institute DB access and model caching to isolate tenant load and reduce repeated setup overhead.
Connection and Workload Control Pool tuning separates control-plane metadata behavior from institute transaction load, reducing contention and improving throughput predictability.
Redis Caching and Safe Invalidation TTL-driven caching, hit/miss metrics, and SCAN-based invalidation were implemented for production-safe cache operations at larger keyspace volume.
Security and Data Protection JWT auth, role-aware access, sanitization, rate limiting, secured S3 object handling with pre-signed URLs, and strict request validation patterns.
Operational Resilience Single VPS with robust clusterization via PM2, dependency-first startup, graceful shutdown ordering, and memory restart boundaries for long-running stability.
Observability-Driven Optimization Prometheus metrics, Loki/Promtail logging, Grafana dashboards, health diagnostics, and cache metrics were integrated for measurable runtime tuning.
